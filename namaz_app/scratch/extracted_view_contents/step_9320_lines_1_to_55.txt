1: # Implementation Plan - Fix Automatic Location & Accurate Nearby Mosques
2: 
3: This plan addresses two user requests:
4: 1. **Automatic location detection is not working (`otomatik konum bulma çalışmıyor`):** In all Flutter screens (`main_screen.dart`, `onboarding_screen.dart`, `settings_screen.dart`), the app was querying `Geolocator.getLastKnownPosition()` first. If a cached location (like Gaziantep) existed, it was immediately used and the app never requested a fresh GPS coordinate. Additionally, Dart's `.toLowerCase()` has a known bug in Turkish where `'İ'.toLowerCase()` produces a combining dot character (`i\u0307`), preventing correct city matching.
5: 2. **Exact location down to the street (`tam konumunu alsın sokağına kadar`):** We must ensure both Flutter and Web apps request fresh high-accuracy live GPS data and use that to query and sort nearby mosques from the Overpass API.
